# Obsidian Writer & Knowledge Architect Skill v2.0

> Claude Skill로 구현한 Obsidian 볼트 기반 글쓰기(책 집필), 기획, 지식 관리 자동화.
> Claude Desktop / Cowork Mode에서 로컬 Obsidian MCP 서버와 함께 사용합니다.

**[English README](README.md)**

## 이 스킬이 다른 것들과 다른 점

대부분의 Obsidian MCP 도구(cyanheads, MarkusPfundstein, bitbonsai 등)는 **기본적인 볼트 조작** — 읽기, 쓰기, 검색, 삭제에 집중합니다. 배관은 제공하지만 지능은 제공하지 않습니다. 마찬가지로, 기존 Claude 글쓰기 스킬들은 **볼트와 무관**합니다 — 축적된 지식을 활용하지 않고 처음부터 콘텐츠를 생성합니다.

**obsidian-writer-skill은 이 간극을 연결합니다.** 볼트를 단순한 파일 시스템이 아닌 살아있는 지식 베이스로 활용하는 9개의 지능형 워크플로우를 제공합니다.

### 기존 접근 방식과의 비교

| 측면 | 다른 MCP 서버 | 범용 글쓰기 스킬 | **이 스킬** |
|------|-------------|----------------|------------|
| 볼트 상호작용 | CRUD 작업 | 없음 | **지능형 다층 검색 + AI 추론** |
| 글쓰기 지원 | 없음 | 범용 프롬프트 | **볼트 자료 기반 초안 작성 + 자동 참조** |
| 지식 재활용 | 수동 검색 | 없음 | **관련 과거 노트 자동 발견** |
| 품질 관리 | 없음 | 기본 문법 | **팩트체크 + AI 글투 제거 + 출처 검증** |
| 정리/분류 | 기본 폴더/태그 | 없음 | **AI 기반 자동 분류 + 계층적 태깅** |
| 메모리 | 상태 비저장 | 상태 비저장 | **프로젝트 메모리: 교훈 기록 + 의사결정 로그** |
| 접근 방식 | 도구 레벨 | 프롬프트 레벨 | **워크플로우 레벨 (9개 상호 연결 프로세스)** |

### 핵심 혁신: 시뮬레이션 시맨틱 검색

현재 MCP 서버들은 키워드 매칭만 지원합니다. 고급 대안(Claudesidian/Nexus)은 벡터 데이터베이스와 임베딩 API가 필요합니다. 이 스킬은 Claude의 추론 능력만으로 **90% 이상의 시맨틱 검색 효과**를 달성합니다:

1. **동의어 확장**: "수익" → "수익, 매출, revenue, 이익, 손익"
2. **개념 계층 탐색**: "인공지능" → "머신러닝, 딥러닝, LLM, 신경망"
3. **이중 언어 검색**: 한국어 + 영어 키워드 동시 검색
4. **맥락 추론**: "사업계획" → "BMC, 타당성, 시장분석, 투자"

추가 설정, API 비용, 임베딩 모델이 필요 없습니다.

## 9개 워크플로우

| # | 워크플로우 | 기능 |
|---|----------|------|
| 1 | **스마트 리서치** | 주제 분해 → 다층 볼트 검색 → 관련성 등급(A/B/C) → 소스맵 생성 |
| 2 | **책 집필** | 목차 설계 → 볼트 노트-챕터 매핑 → 참조 기반 초안 작성 → 진행 추적 |
| 3 | **자동 분류** | 볼트 진단 → 분류 체계 설계 → 태그 표준화 → 일괄 적용 |
| 4 | **기획서 빌더** | 볼트 지식 수집 → 공백 식별 → 구조화 → 기획서 초안 생성 |
| 5 | **일일 큐레이션** | 새 노트 확인 → 연관 관계 발견 → 자동 태깅 → 일일 요약 생성 |
| 6 | **팩트체크** | 주장 추출 → 볼트 내 교차 검증 → 외부 검증 → 신뢰도 등급 부여 |
| 7 | **AI 글투 제거** | AI 패턴 감지(한/영) → 심각도 분류 → 자연스러운 대안 제시 |
| 8 | **디지털 아카이브** | 엔티티 추출 → 관계 매핑 → 지식 그래프 구축 → 볼트 건강 관리 |
| 9 | **프로젝트 메모리** | 의사결정 기록 → 교훈 캡처 → 경험 기반 조언 제공 |

### 워크플로우 연계

9개 워크플로우는 상호 연결됩니다. 리서치(W1)가 집필(W2)로 이어지고, 집필은 팩트체크(W6)와 AI 글투 제거(W7)를 트리거합니다. 분류(W3)는 아카이브(W8)로 연결됩니다. 완료된 모든 프로젝트는 메모리(W9)를 생성하여 향후 작업에 반영됩니다.

## 설치

### 사전 요구사항
- 로컬 MCP 서버가 연결된 Obsidian 볼트
- Claude Desktop 또는 Cowork Mode

### 설정
`obsidian-writer-skill/` 폴더를 스킬 디렉토리에 복사합니다:
```
~/.skills/skills/obsidian-writer/
```

### 파일 구조
```
obsidian-writer-skill/
├── SKILL.md                     # 핵심: 9개 워크플로우 정의
├── README.md                    # 영문 문서
├── README_KR.md                 # 한국어 문서
└── references/
    ├── upgrade-guide.md         # MCP 서버 비교 & 업그레이드 로드맵
    ├── writing-workflows.md     # 상세 집필/기획 가이드
    └── v2-upgrade-analysis.md   # v1→v2 요소 채택/배제 분석
```

### 트리거 키워드
다음 키워드로 스킬이 자동 활성화됩니다: `옵시디안`, `볼트`, `책 집필`, `원고`, `목차`, `노트 검색`, `자동 태그`, `분류`, `팩트체크`, `출처 검증`, `AI 글투`, `글쓰기 탈독소`, `프로젝트 메모리`, `교훈 기록`
