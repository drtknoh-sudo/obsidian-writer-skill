# Obsidian Writer Skill v2.0 업그레이드 적합성 분석

## 제안 요소별 채택/배제 판단

### ✅ 채택 (6개 요소)

| 요소 | 원본 | 적용 방식 | 채택 이유 |
|------|------|----------|----------|
| **source-verification** | 저널리즘 | Workflow 6으로 통합 | 비허구 글쓰기의 핵심. SIFT 방법론의 출처 검증 개념을 볼트 교차 검증 + 외부 검증으로 재설계 |
| **fact-check-workflow** | 저널리즘 | Workflow 6으로 통합 | 주장 추출→근거 수집→등급 산정 프로세스는 기획서/보고서 품질에 직결 |
| **ai-writing-detox** | 글쓰기 품질 | Workflow 7로 독립 | AI 도구 활용이 늘면서 필수적. 한국어/영어 금지 패턴 DB가 실용적 가치 높음 |
| **digital-archive** | 학술/연구 | Workflow 8로 통합 | 엔티티 추출→관계 매핑→지식 그래프는 볼트의 장기 가치를 극대화 |
| **project-memory** | 글쓰기 품질 | Workflow 9로 독립 | 프로젝트 교훈 캡처는 반복 실수 방지와 경험 누적에 핵심적 |
| **academic-writing 일부** | 학술/연구 | 기존 Workflow 2에 부분 반영 | 체계적 글쓰기 구조(IMRaD), 참조 관리 개념을 집필 템플릿에 반영 |

### ❌ 배제 (10개 요소)

| 요소 | 배제 이유 |
|------|----------|
| **foia-requests** | 미국 정보공개청구 전용. 볼트 기반 글쓰기/기획과 무관. 사용자 맥락(한국/오만 기반)에도 맞지 않음 |
| **data-journalism** | 뉴스룸 전용 데이터 파이프라인. 스킬의 핵심인 볼트 활용 글쓰기와 무관. 별도 스킬이 더 적합 |
| **social-media-intelligence** | OSINT, 소셜미디어 모니터링은 볼트 지식 관리와 다른 영역. 스킬 범위를 불필요하게 확장 |
| **newsroom-style** | AP 스타일 표기법은 저널리즘 전용. AI 글투 제거(W7)가 더 범용적이고 실용적 |
| **interview-prep / transcription** | 인터뷰 전사는 별도 도구(Whisper 등)가 필요. 현재 MCP 도구로 처리 불가 |
| **story-pitch / crisis-comms** | 뉴스룸 전용. 책 집필/기획 워크플로우에 통합 시 초점이 흐려짐 |
| **content-access** | Unpaywall/Semantic Scholar API 직접 호출 필요. Cowork 환경에서 실행 제약 많음. Consensus MCP와 WebSearch로 대체 가능 |
| **web-archiving** | Wayback Machine 통합은 외부 의존성 높음. 볼트 자체의 아카이빙(W8)이 더 적합 |
| **page-monitoring** | 웹페이지 변경 감지는 상시 실행 프로세스 필요. 스킬의 on-demand 특성과 충돌 |
| **project-retrospective** | project-memory(W9)에 이미 회고 기능 포함. 별도 분리하면 중복 |

## 채택 시 배제 기준

다음 기준에 해당하는 요소는 과감히 배제했다:

1. **외부 의존성 과다**: 별도 API 키, 상시 실행 프로세스, 외부 서비스가 필요한 요소
2. **MCP 도구 범위 초과**: 현재 Obsidian MCP 10개 도구로 구현 불가능한 기능
3. **사용자 맥락 불일치**: DR. TK의 글쓰기/기획 중심 작업 패턴과 맞지 않는 요소
4. **초점 희석**: 핵심인 "볼트 기반 글쓰기/기획/지식관리"에서 벗어나는 요소
5. **중복**: 기존 워크플로우에서 이미 다루는 기능

## 채택 시 우선 기준

다음 기준에 높은 점수를 받은 요소를 채택했다:

1. **볼트 활용 시너지**: 기존 MCP 도구만으로 구현 가능하며 볼트의 가치를 높이는가
2. **글쓰기/기획 직결성**: 책 집필이나 기획서 작성 품질에 직접 기여하는가
3. **누적 효과**: 시간이 갈수록 볼트가 더 스마트해지는 데 기여하는가
4. **실행 용이성**: 사용자가 자연어 한 줄로 트리거할 수 있는가

## v1.0 → v2.0 변경 요약

| 항목 | v1.0 | v2.0 |
|------|------|------|
| 워크플로우 수 | 5개 | 9개 |
| 팩트체크 | 없음 | 내부 교차검증 + 외부 검증 |
| AI 글투 관리 | 없음 | 금지 패턴 DB + 4가지 톤 옵션 |
| 아카이빙 | 없음 | 엔티티 추출 + 지식 그래프 + 아카이브 규칙 |
| 프로젝트 메모리 | 없음 | 교훈 캡처 + 경험 기반 조언 |
| 신뢰도 관리 | 없음 | 검증 등급 체계 + trust 태그 |
| 집필 템플릿 | 기본 | 팩트체크 로그, confidence 필드 추가 |
| 워크플로우 연계 | 개별 실행 | 9개 워크플로우 연계 맵 제공 |
| 트리거 키워드 | 12개 | 18개 (팩트체크, 출처검증, AI글투, 프로젝트정리 추가) |
