# 글쓰기 & 기획 특화 워크플로우 상세 가이드

## 1. 책 집필 워크플로우

### 1.1 프로젝트 시작: "볼트 기반 집필 준비"

**사용 예시 프롬프트**:
```
"AI 디스토피아 소설 집필을 위해 볼트에서 관련 자료를 모아줘"
"지혜의 시대 개정판 3장을 쓰기 위한 자료를 찾아줘"
```

**Claude가 수행하는 작업**:
1. 주제 관련 키워드 생성 (한글/영문)
2. search_notes → full_text_search → 폴더별 list_notes 순차 실행
3. 각 발견 노트를 get_note로 읽고 관련성 평가
4. A/B/C 등급 분류한 자료 맵 제시
5. 사용자 확인 후, 집필 프로젝트 폴더에 자료 맵 노트 저장

### 1.2 목차 설계: "볼트 자료 기반 목차 생성"

**프로세스**:
1. 자료 맵의 A등급 자료를 중심으로 논리 흐름 설계
2. 각 챕터에 매핑되는 참조 노트 연결
3. 빈 챕터(자료 부족) 식별
4. 목차를 Obsidian 노트로 저장

**목차 노트 예시**:
```markdown
---
created: 2026-03-10
project: AI 디스토피아
type: outline
status: draft
tags:
  - project/writing
  - type/outline
---

# AI 디스토피아 - 목차

## Part 1: 현실의 균열
### Ch.1 - 일상의 변화
- 참조: [[인공지능으로 인한 시장 변화에 따른 기획 분석]]
- 참조: [[인공지능의 발전이 고용에 미치는 영향 분석]]

### Ch.2 - 기술의 이면
- 참조: [[생성형 AI 환각증세(Hallucination) 분석 및 미래 전망 보고서]]
- ⚠️ 추가 자료 필요: 기술 실패 사례
```

### 1.3 챕터 집필: "노트 기반 초안 작성"

**각 챕터에 대해**:
1. 매핑된 참조 노트 내용 수집
2. 핵심 인사이트/팩트 추출
3. 서사 구조에 맞게 재구성
4. 초안 작성 → `create_note`로 저장
5. frontmatter에 상태/출처 기록

### 1.4 교정 및 개선

**프로세스**:
1. 기존 초안 `get_note`로 읽기
2. 볼트에서 보강 자료 추가 검색
3. 논리 흐름, 팩트 검증, 문체 일관성 점검
4. 개선된 버전을 `update_note`로 저장
5. frontmatter status를 `review`로 변경

## 2. 기획서 작성 워크플로우

### 2.1 지식 수집형 기획서

**사용 예시 프롬프트**:
```
"AIDX Modeling 관련 자료를 모아서 사업 타당성 분석 기획서를 만들어줘"
"DCC Sampo 프로젝트 관련 모든 노트를 종합해서 투자 제안서를 만들어줘"
```

**프로세스**:
1. 프로젝트명으로 search_notes → 관련 노트 전수 조사
2. 태그별, 폴더별 추가 탐색
3. 각 노트의 핵심 정보 추출
4. 기획서 구조에 맞게 재배치
5. 빈 섹션에 대한 분석/추론 추가
6. 완성된 기획서를 볼트에 저장

### 2.2 아이디어 발전형 기획서

**프로세스**:
1. 볼트의 관련 아이디어 메모 수집 (`DRTK_2026/AIDX 사업화 아이디어` 등)
2. 기존 분석 보고서에서 데이터 추출
3. 아이디어들을 논리적 구조로 통합
4. SWOT, BMC 등 프레임워크 적용
5. 기획서 초안 저장

## 3. 자동 분류 & 태그 워크플로우

### 3.1 일괄 분류 작업

**사용 예시 프롬프트**:
```
"Obsidian 2025 폴더의 노트를 주제별로 분류하고 태그를 달아줘"
"태그가 없는 노트들을 찾아서 자동으로 태그를 부여해줘"
```

**프로세스**:
1. 대상 폴더의 노트 목록 수집
2. 각 노트 내용 분석 (get_note)
3. AI 기반 카테고리 판별:
   - 주제 분석 (AI, 교육, 사업, 기술 등)
   - 문서 유형 판별 (보고서, 메모, 기획서 등)
   - 프로젝트 연관성 분석
4. 표준화된 태그 생성
5. frontmatter에 태그 추가 (update_note)
6. 분류 결과 리포트 생성

### 3.2 태그 체계 정비

**현재 문제점**:
- 숫자 태그 (#1, #2...) - 의미 없는 태그 다수
- 색상 코드 태그 (#FFFFFF 등) - 실수로 생성된 태그
- 한영 혼재, 계층 구조 없음

**정비 방법**:
1. get_tags로 현재 태그 전수 조사
2. 의미 있는 태그와 노이즈 태그 분리
3. 계층적 태그 체계 설계
4. 각 노트의 태그를 표준화된 체계로 업데이트

## 4. 자료 찾기 고급 전략

### 4.1 시맨틱 검색 시뮬레이션

현재 MCP는 키워드 검색만 지원하므로, Claude의 추론으로 시맨틱 검색을 시뮬레이션:

1. **동의어 확장**: "수익" → "수익, 매출, revenue, 이익, 손익"
2. **상위/하위 개념**: "인공지능" → "머신러닝, 딥러닝, LLM, AI"
3. **관련 맥락어**: "사업계획" → "BMC, 타당성, 시장분석, 투자"
4. **다국어 검색**: 한글 키워드 + 영문 키워드 병행

### 4.2 연관 노트 발견

특정 노트와 관련된 다른 노트를 찾는 전략:

1. 대상 노트의 핵심 키워드 추출 (get_note)
2. 추출된 키워드로 다층 검색
3. 같은 폴더의 다른 노트 탐색
4. 같은 태그를 가진 다른 노트 식별
5. 발견된 관련 노트 목록 제시

### 4.3 시간순 맥락 추적

특정 주제가 볼트에서 어떻게 발전해왔는지 추적:

1. 주제 관련 노트를 모두 수집
2. modified 날짜순 정렬
3. 시간순 사고 흐름 재구성
4. 아이디어 발전 과정 시각화
