# Obsidian Writer & Knowledge Architect Skill v2.0

## Overview
이 스킬은 Obsidian 볼트에 저장된 지식을 최대한 활용하여 글쓰기(특히 책 집필)와 기획 업무의 생산성을 극대화하는 종합 워크플로우를 제공한다. 단순한 노트 읽기/쓰기를 넘어, 볼트 전체를 '지능형 제2의 뇌'로 활용하는 체계적 방법론이다.

v2.0 신규: 출처 검증, 팩트체크, AI 글투 제거, 디지털 아카이빙, 프로젝트 메모리 워크플로우가 추가되었다.

## When This Skill Triggers
아래 조건 중 하나 이상 해당 시 자동 트리거:
- Obsidian 볼트의 노트를 활용한 글쓰기/집필 요청
- 볼트 내 자료 검색, 분류, 태그 관리 요청
- 과거 저장 정보에서 특정 주제 관련 자료 찾기
- 기획안/제안서 작성 시 볼트 지식 활용 요청
- 노트 자동 분류, 메타데이터 정리 요청
- 팩트체크, 출처 검증, 글쓰기 품질 개선 요청
- 프로젝트 회고, 교훈 기록, 지식 캡처 요청
- "옵시디안", "볼트", "노트에서 찾아", "관련 자료", "집필", "원고", "팩트체크", "출처 검증" 키워드

## Core Workflows (기존 5개 + 신규 4개)

### Workflow 1: 지능형 자료 탐색 (Smart Research)
책 집필이나 기획서 작성 시, 볼트에 이미 있는 관련 자료를 체계적으로 찾아낸다.

**Step 1: 주제 분해**
사용자의 요청 주제를 핵심 키워드, 관련 개념, 유사어로 분해한다.

```
예시: "AI 시대의 인성교육"
→ 키워드: 인공지능, AI, 인성교육, 가치관, 교육
→ 관련 개념: 디스토피아, 윤리, 인간성, 미래사회
→ 유사어: 인공지능 시대, AI era, 교육 혁신
```

**Step 2: 다층 검색**
분해된 키워드를 활용해 3단계로 검색한다:

1. **제목 검색** (`search_notes`): 각 키워드로 제목 매칭
2. **전문 검색** (`full_text_search`): 본문에서 키워드 탐색
3. **폴더 탐색** (`list_notes` with folder): 관련 폴더의 전체 노트 스캔

**Step 3: 관련성 평가**
검색 결과를 읽고(`get_note`) 관련성을 A/B/C 등급으로 분류:
- **A등급**: 직접 인용/참조 가능한 핵심 자료
- **B등급**: 배경 지식이나 보조 자료
- **C등급**: 간접적 연관만 있는 참고 자료

**Step 4: 자료 맵 생성**
발견한 자료를 구조화된 맵으로 정리:

```markdown
## 자료 맵: [주제]
### A등급 (핵심 자료)
- [[노트명]] - 핵심 내용 요약 (1줄)
### B등급 (보조 자료)
- [[노트명]] - 활용 포인트
### C등급 (참고)
- [[노트명]] - 연관 키워드
### 🔍 볼트 외부 보완 필요 영역
- [주제 X] - 볼트에 관련 자료 없음, 외부 리서치 권장
```

### Workflow 2: 책 집필 지원 (Book Writing Assistant)
볼트의 자료를 기반으로 체계적인 집필 워크플로우를 제공한다.

**Phase 1: 구조 설계**
1. 볼트 전체 스캔하여 주제 관련 기존 자료 파악
2. 목차 초안 생성 → 각 챕터에 관련 노트 매핑
3. 자료 부족 챕터 식별 → 추가 리서치 필요 영역 표시

**Phase 2: 챕터별 집필**
각 챕터에 대해:
1. 관련 노트 내용 수집 (`get_note`)
2. 핵심 논점/사실 추출 및 정리
3. **[v2.0] AI 글투 필터 적용** (아래 Workflow 7 참조)
4. 초안 작성 → Obsidian 노트로 저장 (`create_note`)
5. 저장 경로: `{프로젝트폴더}/원고/Ch{번호}_{제목}.md`

**Phase 3: 교차 참조 및 팩트 검증**
1. 각 챕터 간 연결고리 분석
2. 볼트 내 추가 근거자료 검색
3. **[v2.0] 핵심 주장에 대한 팩트 체크** (아래 Workflow 6 참조)
4. 인용/참조 표시 추가

**집필 노트 템플릿**:
```markdown
---
created: {date}
modified: {date}
project: {책 제목}
chapter: {챕터 번호}
status: draft | review | final
confidence: high | medium | low
tags:
  - 집필
  - {주제태그}
sources:
  - "[[관련노트1]]"
  - "[[관련노트2]]"
fact_check_status: pending | verified | flagged
---

# {챕터 제목}

## 핵심 논점
> [이 챕터에서 전달할 핵심 메시지]

## 본문

## 참조 자료
- [[노트명]] - 활용한 내용 요약

## 팩트 검증 로그
| 주장 | 근거 | 검증 상태 |
|------|------|-----------|

## 수정 메모
-
```

### Workflow 3: 자동 분류 및 태그 관리 (Auto-Classification)
볼트의 노트를 체계적으로 분류하고 태그를 자동 부여한다.

**Step 1: 볼트 현황 진단**
1. `get_vault_info`: 전체 규모 파악
2. `list_folders`: 폴더 구조 분석
3. `get_tags`: 기존 태그 체계 분석
4. `list_notes`: 미분류 노트 식별

**Step 2: 분류 체계 설계**
현재 볼트 구조를 분석하여 최적의 분류 체계를 제안:

```
분류 기준:
1. 프로젝트별: AIDX, DCC Sampo, Trutha, 집필 등
2. 유형별: 기획서, 분석보고서, 아이디어메모, 학습노트, 회의록
3. 상태별: 초안, 진행중, 완료, 보관
4. 주제별: AI, 교육, 사업, 기술, 인문학
5. [v2.0] 신뢰도별: 검증됨, 미검증, 외부인용
```

**Step 3: 태그 체계 표준화**
일관된 태그 체계를 설계하고 적용:

```yaml
# 태그 계층 구조
프로젝트:
  - project/aidx
  - project/trutha
  - project/writing
유형:
  - type/report
  - type/plan
  - type/memo
  - type/research
  - type/archive         # [v2.0] 아카이브 유형 추가
상태:
  - status/draft
  - status/active
  - status/archive
주제:
  - topic/ai
  - topic/education
  - topic/business
신뢰도:                    # [v2.0] 신규
  - trust/verified
  - trust/unverified
  - trust/external-cite
```

**Step 4: 자동 태그 부여**
각 노트의 내용을 분석하여 적절한 태그를 frontmatter에 추가:
1. `get_note`로 내용 읽기
2. 내용 분석하여 카테고리 판별
3. `update_note`로 태그 추가된 frontmatter 업데이트

### Workflow 4: 기획서 작성 지원 (Planning Document Builder)
볼트의 지식을 기반으로 기획서/제안서를 작성한다.

1. **지식 수집**: 주제 관련 볼트 내 모든 자료 탐색
2. **Gap 분석**: 기획서에 필요하지만 볼트에 없는 정보 식별
3. **구조화**: 수집된 정보를 기획서 구조에 맞게 재배치
4. **[v2.0] 출처 검증**: 핵심 데이터/주장의 출처 신뢰도 평가
5. **초안 생성**: 볼트 자료 + 분석을 결합한 기획서 초안
6. **볼트 저장**: 결과물을 적절한 위치에 노트로 저장

---

### Workflow 5: 일일 지식 정리 (Daily Knowledge Curation)
매일의 정보를 체계적으로 정리하고 기존 지식과 연결한다.

1. 새로 입력된 노트/클리핑 확인
2. 기존 노트와의 연관성 분석
3. 자동 태그 부여 및 폴더 분류 제안
4. 관련 기존 노트 목록 제시
5. 데일리 노트에 요약 기록

## v2.0 신규 Workflows

### Workflow 6: 출처 검증 & 팩트체크 (Source Verification & Fact-Check)
볼트의 노트에 담긴 주장, 데이터, 인용의 신뢰도를 체계적으로 검증한다.
비허구 글쓰기(보고서, 기획서, 논픽션 집필)의 품질을 근본적으로 높이는 핵심 워크플로우.

**When to Use**:
- 집필 중인 원고의 핵심 주장이 사실에 기반하는지 확인할 때
- 기획서/보고서의 데이터나 통계의 출처를 검증할 때
- "이 노트의 내용이 맞는지 확인해줘" 요청 시
- 볼트의 오래된 노트 정보가 여전히 유효한지 점검할 때

**Step 1: 주장 추출 (Claim Extraction)**
대상 노트를 `get_note`로 읽고, 팩트 체크가 필요한 주장을 식별한다:

```
추출 기준:
- 구체적 수치/통계 ("시장 규모 XX억 달러")
- 인과관계 주장 ("X 때문에 Y가 발생")
- 시간적 주장 ("2025년부터 시행")
- 인용 ("~에 따르면")
- 비교/순위 ("세계 1위", "가장 빠른")
```

**Step 2: 볼트 내 교차 검증 (Internal Cross-Check)**
동일 주제를 다룬 다른 노트에서 일관성을 확인한다:

1. 주장의 핵심 키워드로 `full_text_search` 실행
2. 같은 주제의 다른 노트에서 동일/유사 주장 탐색
3. 볼트 내 모순되는 정보 식별
4. 시간순 비교: 더 최신 노트의 정보가 업데이트되었는지 확인

**Step 3: 외부 검증 (External Verification)**
`WebSearch` 도구를 활용한 핵심 주장 검증:

1. 주요 주장을 검색 쿼리로 변환
2. 신뢰도 높은 출처(학술, 정부, 공신력 있는 매체) 우선 확인
3. 학술 데이터가 필요하면 `consensus_search` 도구 활용
4. 검증 결과를 등급화:
   - ✅ **Verified**: 복수 신뢰 출처에서 확인
   - ⚠️ **Partially Verified**: 일부만 확인 또는 약간의 차이 존재
   - ❌ **Unverified/Disputed**: 확인 불가 또는 상충 정보 발견
   - 🔄 **Outdated**: 원본은 맞았으나 최신 정보로 업데이트 필요

**Step 4: 검증 보고서 생성**
검증 결과를 노트로 기록한다:

```markdown
---
created: {date}
type: fact-check
target: "[[검증 대상 노트명]]"
tags:
  - type/fact-check
  - trust/verified
---

# 팩트체크: {대상 노트 제목}

## 검증 요약
- 검증 일시: {date}
- 총 주장 수: X개
- 검증 완료: Y개 (✅ Z개, ⚠️ W개, ❌ V개)

## 상세 검증 결과

### ✅ 검증 완료
| 주장 | 근거 출처 | 비고 |
|------|----------|------|

### ⚠️ 부분 검증
| 주장 | 확인된 내용 | 차이점 |
|------|------------|--------|

### ❌ 미검증/반박
| 주장 | 문제점 | 수정 권고 |
|------|--------|----------|

### 🔄 업데이트 필요
| 주장 | 원본 시점 | 최신 정보 |
|------|----------|----------|
```

**Step 5: 원본 노트 반영**
사용자 확인 후, 검증 결과를 원본 노트의 frontmatter에 반영:
- `fact_check_status: verified` 또는 `flagged`
- `fact_check_date: {date}`
- `fact_check_note: "[[팩트체크 노트 링크]]"`

### Workflow 7: AI 글투 제거 & 문체 다듬기 (AI Writing Detox)
AI가 생성한 텍스트의 전형적 패턴을 감지하고 자연스러운 문체로 개선한다.
책 집필, 기획서 등 최종 산출물의 품질을 전문가 수준으로 끌어올린다.

**When to Use**:
- AI 도움으로 작성한 초안을 자연스럽게 다듬을 때
- "이 글이 AI가 쓴 것 같아 보이지 않게 해줘" 요청 시
- 최종 원고의 문체를 통일하고 싶을 때
- 집필 Phase 2에서 자동으로 적용됨

**감지 대상 패턴 (AI 글투 금지어/패턴)**:

```
[한국어 AI 패턴]
- "~의 중요성은 아무리 강조해도 지나치지 않습니다"
- "다양한 관점에서 살펴보면"
- "이러한 맥락에서 볼 때"
- "종합적으로 고려할 때"
- "~은 매우 중요한 역할을 합니다"
- "~에 대한 심층적인 이해가 필요합니다"
- 과도한 "또한", "더불어", "나아가" 나열
- 불필요한 서론-본론-결론 선언 ("먼저~, 다음으로~, 마지막으로~")
- 모든 문단을 결론으로 끝내는 패턴
- "획기적인", "혁신적인", "패러다임 전환" 남발

[영문 AI 패턴]
- "It's important to note that..."
- "In today's rapidly evolving landscape..."
- "Delve into", "Navigate the complexities"
- "Leverage", "Synergy", "Holistic approach"
- "At the end of the day"
- Excessive hedging: "It could be argued that..."
```

**개선 프로세스**:

1. **스캔**: 대상 노트를 `get_note`로 읽고 AI 패턴 감지
2. **분류**: 각 패턴을 심각도별로 분류
   - 🔴 **즉시 수정**: 독자가 바로 AI 생성임을 알아챌 패턴
   - 🟡 **권장 수정**: 문체 개선이 바람직한 패턴
   - 🟢 **참고**: 맥락에 따라 자연스러울 수 있는 패턴
3. **대안 제시**: 각 패턴에 대한 자연스러운 대안 3가지 이상 생성
4. **적용**: 사용자 확인 후 `update_note`로 개선된 버전 저장

**문체 톤 옵션**:
사용자 선호에 따라 4가지 톤 중 선택:
- **학술/전문**: 정확하고 절제된 표현, 주장-근거 구조
- **에세이/수필**: 1인칭 관점, 개인적 통찰, 서사적 흐름
- **저널리즘**: 팩트 중심, 간결, 5W1H
- **대화체**: 친근하고 이해하기 쉬운 표현

### Workflow 8: 디지털 아카이빙 & 지식 그래프 (Digital Archive & Knowledge Graph)
볼트를 체계적인 디지털 아카이브로 발전시키고, 노트 간 관계를 지식 그래프로 관리한다.

**When to Use**:
- "볼트를 정리해서 체계적인 아카이브로 만들어줘"
- "이 프로젝트의 관련 노트 관계도를 그려줘"
- 대량의 노트를 주제별로 체계화할 때
- 프로젝트 간 교차점을 발견하고 싶을 때

**Step 1: 엔티티 추출 (Entity Extraction)**
볼트의 노트들을 스캔하여 핵심 엔티티를 추출한다:

```
추출 대상:
- 인물: 이해관계자, 파트너, 전문가
- 조직: 회사, 기관, 정부 부처
- 프로젝트: AIDX, DCC Sampo, Trutha 등
- 기술: AI, LLM, 벡터DB 등
- 개념: 소버린 AI, 인성교육, 디스토피아 등
- 장소: 오만, 무스카트, 서울 등
```

**Step 2: 관계 매핑 (Relationship Mapping)**
추출된 엔티티 간 관계를 분석한다:

1. 같은 노트에 등장하는 엔티티들의 관계 추론
2. 프로젝트-인물-기술-개념 간 연결 생성
3. 시간순 관계 변화 추적

**Step 3: 지식 그래프 노트 생성**
분석 결과를 마스터 인덱스 노트로 저장:

```markdown
---
created: {date}
type: knowledge-graph
scope: vault-wide
tags:
  - type/archive
  - type/index
---

# 볼트 지식 그래프

## 프로젝트 허브
### AIDX Modeling
- 관련 인물: {list}
- 핵심 기술: {list}
- 관련 노트: {count}개
- 연결 프로젝트: DCC Sampo (기술 공유), Trutha (데이터 연계)

### DCC Sampo
- 관련 인물: {list}
- 핵심 기술: {list}
- 관련 노트: {count}개
- 연결 프로젝트: AIDX (기술 공유)

## 주제 클러스터
### AI & 기술
- 핵심 노트: [[list]]
- 교차 프로젝트: AIDX, DCC Sampo, Trutha
### 교육 & 인문학
- 핵심 노트: [[list]]
- 교차 프로젝트: 집필, 인성교육
```

**Step 4: 아카이브 규칙 적용**
시간이 지나면서 볼트를 건강하게 유지하는 규칙:

1. **신선도 평가**: 6개월 이상 미수정 노트 → 재검증 필요 마킹
2. **중복 감지**: 유사 제목/내용의 노트 → 통합 제안
3. **고아 노트 식별**: 어떤 프로젝트에도 연결되지 않은 노트 → 분류 제안
4. **아카이브 이동**: 완료된 프로젝트 노트 → archive 폴더로 이동 제안

### Workflow 9: 프로젝트 메모리 & 회고 (Project Memory & Retrospective)
진행 중이거나 완료된 프로젝트의 핵심 지식, 교훈, 의사결정 이력을 체계적으로 캡처한다.
미래 프로젝트에서 과거 경험을 즉시 활용할 수 있게 만든다.

**When to Use**:
- 프로젝트 마무리 시 교훈을 기록할 때
- "DCC Sampo에서 배운 점을 정리해줘"
- 새 프로젝트 시작 시 과거 유사 프로젝트 경험을 활용할 때
- "이전에 비슷한 일을 했을 때 어떻게 했지?" 질문 시

**Phase 1: 프로젝트 지식 캡처**
프로젝트 관련 노트를 모두 수집하여 핵심 지식을 추출한다:

1. 프로젝트명으로 볼트 전체 검색 (Workflow 1 활용)
2. 노트를 시간순 정렬하여 프로젝트 진행 이력 재구성
3. 핵심 의사결정, 변경사항, 결과를 추출

**Phase 2: 프로젝트 메모리 노트 생성**

```markdown
---
created: {date}
type: project-memory
project: {프로젝트명}
status: active | completed | paused
tags:
  - type/project-memory
  - project/{project-id}
---

# 프로젝트 메모리: {프로젝트명}

## 프로젝트 개요
- 기간: {시작} ~ {종료/현재}
- 목표: {핵심 목표}
- 결과: {현재 상태/성과}

## 핵심 의사결정 로그
| 시점 | 결정 사항 | 이유 | 결과 |
|------|----------|------|------|

## 기술/전략 자산
> 다른 프로젝트에서 재활용 가능한 지식

## 교훈 (Lessons Learned)
### 잘된 점 (Keep)
### 개선할 점 (Improve)
### 시도할 것 (Try)

## 관련 자료 인덱스
- [[노트1]] - 역할/내용 요약
- [[노트2]] - 역할/내용 요약

## 연결 프로젝트
- [[프로젝트 메모리: 다른 프로젝트]] - 연결 이유
```

**Phase 3: 경험 기반 조언 제공**
새 프로젝트 시작 시, 과거 프로젝트 메모리를 검색하여:
1. 유사한 도전과제에서의 경험 찾기
2. 과거 교훈을 기반으로 주의사항 제시
3. 재활용 가능한 전략/템플릿 추천

## Best Practices

### 검색 전략
- **넓게 시작, 좁혀가기**: 먼저 broad한 키워드로 검색 후 정밀 검색
- **복수 키워드**: 하나의 주제에 대해 최소 3-5개 키워드로 검색
- **폴더 + 검색 병행**: 특정 폴더 탐색과 전체 검색을 함께 사용
- **한영 병행**: 한글과 영문 키워드 모두 사용

### 글쓰기 원칙
- 원본 노트를 직접 수정하지 않고, 새 노트에 가공된 내용 저장
- 출처 노트를 항상 링크로 참조 (`[[노트명]]`)
- 집필 중간 결과물은 별도 프로젝트 폴더에 저장
- 초안 → 검토 → 최종의 상태 관리를 frontmatter로 추적
- **[v2.0] AI 글투 필터를 최종 교정 단계에서 반드시 적용**
- **[v2.0] 핵심 주장은 팩트체크 워크플로우로 검증 후 게시**

### 태그 관리
- 계층적 태그 사용 (`topic/ai`, `project/aidx`)
- 최소 2개, 최대 7개의 태그 부여
- 분기별 태그 리뷰 및 통합 정리
- **[v2.0] 신뢰도 태그(`trust/`)를 팩트체크 완료 노트에 추가**

### 프로젝트 메모리 원칙
- **[v2.0] 프로젝트 마일스톤마다 메모리 노트 업데이트**
- **[v2.0] 새 프로젝트 시작 시 유사 프로젝트 메모리 먼저 검색**
- **[v2.0] 교훈은 구체적 행동으로 기록 (추상적 반성 X)**

## DR. TK 볼트 특화 설정

```
총 333개 노트, 53개 폴더
```

```
[자료 탐색 W1] ──→ [책 집필 W2] ──→ [AI글투 제거 W7]
      │                  │                    │
      │                  ▼                    ▼
      │          [팩트체크 W6] ──→ [프로젝트 메모리 W9]
      │
      ▼
[자동 분류 W3] ──→ [아카이빙 W8]
      │
      ▼
[기획서 작성 W4] ──→ [팩트체크 W6]
      │
      ▼
```
